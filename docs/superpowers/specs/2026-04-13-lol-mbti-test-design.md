# LoL MBTI 性格测试网站 — 设计文档

## 概述

一个基于符文大陆世界观的性格测试网站。用户通过 32 道题目（24 道量表题 + 8 道情景题）测试自己在 8 个自定义性格维度上的倾向，生成雷达图，匹配最相似的 LoL 城邦和英雄，并输出一段丰富的性格总结。

- **技术栈**：Nuxt 4（Vue 3），纯静态部署，全客户端计算
- **视觉风格**：符文大陆奇幻风——羊皮纸纹理、古老符文装饰、手绘质感
- **目标时长**：5-8 分钟完成测试

---

## 一、8 个性格维度

围绕 LoL 符文大陆世界观设计，每个维度是一对对立倾向。雷达图的 8 个顶点对应 8 个维度的高分端，分值范围 1-10。

| # | 维度名 | 低分端 | 高分端 | 说明 |
|---|--------|--------|--------|------|
| 1 | 秩序 ⟷ 混沌 | 规则、纪律、制度 | 自由、叛逆、打破束缚 |
| 2 | 理性 ⟷ 直觉 | 逻辑分析、策略计划 | 感觉驱动、灵感行动 |
| 3 | 独行 ⟷ 羁绊 | 独立自主、单打独斗 | 重视团队、守护同伴 |
| 4 | 和平 ⟷ 征服 | 避免冲突、寻求和谐 | 主动出击、追求力量 |
| 5 | 守护 ⟷ 探索 | 保护已有、坚守阵地 | 冒险未知、追寻新知 |
| 6 | 光明 ⟷ 暗影 | 正义、坦荡、透明 | 隐秘、手段、灰色地带 |
| 7 | 坚韧 ⟷ 灵巧 | 硬扛、正面对决、不屈 | 灵活变通、四两拨千金 |
| 8 | 务实 ⟷ 理想 | 当下利益、实际结果 | 长远愿景、信念追求 |

## 二、题目设计与计分

### 量表题（24 题，每个维度 3 题）

每题是一个陈述句，用户选择符合程度，共 5 个等级：

- 非常不符合 = 1
- 比较不符合 = 2
- 一般 = 3
- 比较符合 = 4
- 非常符合 = 5

部分题目反向计分（描述低分端倾向时）。每个维度 3 题，原始分 3-15，归一化到 1-10。

### 情景题（8 题，每个维度 1 题）

用 LoL 世界观包装的情景选择，每题 4 个选项，每个选项对应不同维度的分值加权。例如：

> **你的船队在比尔吉沃特港外遭遇海雾，前方隐约传来求救声。**
>
> A. 立刻驶向声源，有人需要帮助（和平 +2，羁绊 +1）
> B. 先派斥候探查，确认不是陷阱再说（理性 +2，务实 +1）
> C. 绕道而行，我不会拿船员的命冒险（守护 +2，务实 +1）
> D. 全速冲进去，海雾里说不定有宝藏（探索 +2，混沌 +1）

### 最终计分

量表题得分（权重 70%）+ 情景题得分（权重 30%）→ 每个维度最终分值 1-10。

## 三、城邦与英雄匹配

### 城邦匹配

为每个城邦预设一组"理想维度值"（8 维向量），计算用户维度与各城邦的余弦相似度，取最高的作为匹配城邦。

10 个城邦的核心维度倾向：

| 城邦 | 核心维度倾向 |
|------|-------------|
| 德玛西亚 | 秩序↑ 光明↑ 守护↑ 坚韧↑ |
| 诺克萨斯 | 征服↑ 务实↑ 坚韧↑ 混沌偏中 |
| 艾欧尼亚 | 直觉↑ 和平↑ 理想↑ 灵巧↑ |
| 皮尔特沃夫 | 理性↑ 秩序↑ 探索↑ 务实↑ |
| 祖安 | 混沌↑ 探索↑ 灵巧↑ 暗影↑ |
| 弗雷尔卓德 | 坚韧↑ 征服↑ 独行↑ 务实↑ |
| 比尔吉沃特 | 混沌↑ 灵巧↑ 探索↑ 独行↑ |
| 暗影岛 | 暗影↑ 独行↑ 混沌↑ 理想↑ |
| 恕瑞玛 | 秩序↑ 征服↑ 理想↑ 坚韧↑ |
| 巨神峰 | 理想↑ 光明↑ 守护↑ 直觉↑ |

### 英雄匹配

每个城邦下预设 3-4 个代表英雄，每个英雄有自己的 8 维向量。匹配到城邦后，在该城邦英雄池中再做一次相似度计算，选出最匹配的英雄。

初期总共约 30-40 个英雄。

### 结果展示

- 匹配城邦（名称 + 徽章 + 一句话描述）
- 匹配英雄（名称 + 头衔 + 匹配度百分比）
- Top 3 英雄（主匹配大卡 + 2 个次匹配小卡，可能跨城邦）

## 四、结果页面与可视化

### 雷达图

8 个顶点对应 8 个维度，Canvas 或 SVG 绘制：

- 同心环：3 层（3.3 / 6.6 / 10）作为刻度参考
- 用户多边形：8 个顶点连线，半透明金色填充 + 描边
- 顶点标注：每个顶点外侧标注维度名称和分值
- 动画：从中心向外展开的绘制动画，1-2 秒

### 维度解读

雷达图下方，每个维度一行简短解读。每个分值区间（1-3 / 4-6 / 7-10）对应不同文案，共 8×3 = 24 条。

示例：
> **混沌 8.2** — 你天生排斥规则的束缚，相信真正的力量来自自由意志。

### 英雄结果卡

- 英雄名称 + 头衔
- 城邦归属 + 城邦徽章
- 匹配度百分比
- Top 3 英雄列表（主匹配大卡 + 2 个次匹配小卡）

### 四段式总结（300-500 字）

**第一段 · 核心人格画像**
基于用户 8 个维度中最突出的 2-3 个，用自然语言描绘整体性格轮廓、气质和行为模式。

**第二段 · 城邦归属感**
描述用户为什么属于这个城邦——城邦的文化、价值观和用户性格的共鸣点，用符文大陆世界观包装。

**第三段 · 英雄共鸣**
具体描写用户和匹配英雄在哪些方面相似——决策方式、面对困境的态度、对待同伴的方式，引用英雄背景故事片段来印证。

**第四段 · 光与影**
基于维度中最低的 1-2 项，点出用户的成长空间。用英雄故事的方式表达，给结果一个有深度的收尾。

每段 80-120 字，每段有多套模板文案，根据维度组合动态拼接。

## 五、分享图片生成

点击「保存结果」按钮，使用 html2canvas 在客户端截取隐藏的分享卡片 DOM，生成 PNG 下载。无需后端。

### 分享卡片（750 × 1334 px，竖版）

| 区域 | 内容 |
|------|------|
| 顶部 | 网站标题 + 符文装饰线 |
| 城邦徽章 | 城邦名称 + 小型徽章图标 |
| 迷你雷达图 | 缩小版 8 维雷达图 |
| 英雄信息 | 英雄名称 + 头衔 + 匹配度 |
| 一句话总结 | 从四段总结中提取核心句（30-50 字） |
| 底部 | 网站 URL + 引导文案「来测测你属于哪片土地」 |

分享卡片是隐藏的 `<div>`，固定像素布局确保截图一致。移动端和桌面端均可正常工作。

## 六、页面流程与交互

整个网站是单页应用，5 个全屏场景：

### 1. 首页 / 欢迎页
- 符文大陆风格标题「你属于哪片土地？」
- 羊皮纸质感背景 + 淡金色符文装饰
- 「开始测试」按钮，点击淡入过渡到答题页
- 底部说明：约 5-8 分钟 / 32 道题

### 2. 量表题阶段（24 题）
- 顶部进度条（羊皮纸卷轴展开风格），显示「第 X / 24 题」
- 每题一个画面，陈述句居中
- 5 个选项横排，从「非常不符合」到「非常符合」，大小渐变
- 选中后 0.5 秒过渡到下一题
- 可点击进度条回到之前的题目修改答案

### 3. 情景题阶段（8 题）
- 进度条更新为「情景 X / 8」
- 手绘风格情景描述
- 4 个选项卡片垂直排列
- 选中时卡片发光 + 轻微放大，0.5 秒后下一题

### 4. 计算过渡页
- 「符文正在解读你的命运...」+ 旋转符文动画
- 停留 2-3 秒增加仪式感
- 自动过渡到结果页

### 5. 结果页
- 依次渐入展示：城邦归属 → 雷达图 → 维度解读 → 英雄匹配卡 → 四段总结
- 每个区块随滚动渐入（Intersection Observer）
- 底部：「保存结果」按钮（下载分享图片）+「重新测试」按钮

全程无需刷新页面，Vue Router 管理状态，composable 存储答题数据。

## 七、技术架构

### 目录结构

```
app/
├── app.vue                    # 根组件
├── pages/
│   └── index.vue              # 主页面（管理场景切换）
├── components/
│   ├── WelcomePage.vue        # 欢迎页
│   ├── ScaleQuestion.vue      # 量表题组件
│   ├── ScenarioQuestion.vue   # 情景题组件
│   ├── ProgressBar.vue        # 进度条
│   ├── LoadingTransition.vue  # 计算过渡页
│   ├── ResultPage.vue         # 结果页容器
│   ├── RadarChart.vue         # 雷达图（Canvas）
│   ├── DimensionReadings.vue  # 维度解读列表
│   ├── HeroCard.vue           # 英雄匹配卡片
│   ├── SummaryText.vue        # 四段式总结
│   └── ShareCard.vue          # 分享图片卡片（隐藏）
├── composables/
│   ├── useQuiz.ts             # 答题状态管理（当前题号、答案记录、阶段切换）
│   ├── useScoring.ts          # 计分逻辑（维度分值计算、归一化）
│   └── useMatching.ts         # 匹配逻辑（余弦相似度、城邦/英雄匹配）
├── data/
│   ├── scaleQuestions.ts      # 24 道量表题数据
│   ├── scenarioQuestions.ts   # 8 道情景题数据
│   ├── regions.ts             # 10 个城邦数据（名称、维度向量、描述、徽章）
│   ├── heroes.ts              # 30-40 个英雄数据（名称、头衔、城邦、维度向量、描述）
│   └── summaryTemplates.ts    # 总结模板文案（四段式，按维度组合）
├── assets/
│   ├── fonts/                 # 衬线字体（思源宋体等）
│   ├── textures/              # 羊皮纸纹理、噪点叠加
│   └── icons/                 # 城邦徽章 SVG
└── styles/
    └── global.css             # 全局样式、CSS 变量、字体定义
```

### 关键依赖

- **Nuxt 4**：框架
- **html2canvas**：分享图片生成
- **无其他重型依赖**：雷达图用原生 Canvas API 手写，不引入图表库

### 部署

- `nuxt generate` 生成纯静态文件
- 部署到 Vercel / Netlify / GitHub Pages 均可
